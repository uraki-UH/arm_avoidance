# ToPoFuzzy Viewer & Editor

A high-performance web-based Point Cloud Viewer and Editor built with React, Three.js, and C++ with full ROS2 integration.

## Features
- **High Performance Rendering**: Capable of rendering millions of points using Three.js and custom shaders.
- **ROS2 Integration**: Real-time streaming from ROS2 topics (`/points`, `/topological_map`).
- **Growing Neural Gas (GNG)**: Execute AiS-GNG processing on point cloud layers from the GUI.
- **Advanced Visualization**:
  - Point Cloud streaming (PCD/ROS2)
  - GNG Topology visualization
  - Heatmaps (Height, Distance, Intensity)
  - Clipping Planes
- **Editing Tools**:
  - Selection (Frustum/Rectangle)
  - Point Deletion (Local & Server-synced)
  - Interactive Transform (Translate, Rotate, Scale)
- **File Support**: PCD, LAS, LAZ, PLY

## Getting Started

### Prerequisites
- **Node.js** (v16+)
- **CMake** (v3.16+)
- **C++ Compiler** (C++17 support)
- **ROS2 Humble** (Required)
- **PyTorch** (For ais_gng classification)

### 1. Backend Setup

The backend is a ROS2 workspace. Build with colcon:

```bash
cd backend

# Source ROS2
source /opt/ros/humble/setup.bash

# Build all packages
colcon build --symlink-install

# Source the workspace
source install/setup.bash
```

### 2. Frontend Setup

```bash
cd frontend
npm install
```

## Running the Application

### 1. Start the Backend

```bash
cd backend
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch topo_fuzzy_viewer viewer_stack.launch.py
```
The server will start listening on `ws://localhost:9001`.

### 2. Start the Frontend

```bash
cd frontend
npm run dev
```
Open [http://localhost:5173]in your browser.

### 3. Start GNG Processing (Optional)

```bash
ros2 run ais_gng ais_gng_node
```

## Usage Guide

1.  **Connecting**: Click the **"Connect WebSocket"** button in the sidebar to connect to the backend.
2.  **Streaming**:
    *   Publish PointCloud2 messages to `/points`.
    *   Publish GNG TopologicalMap messages to `/topological_map`.
    *   Publish robot payloads to `/viewer/internal/stream/robot`.
    *   The viewer will automatically stream and visualize the data.
3.  **File Import**: Drag & Drop `.pcd`, `.las`, `.laz`, or `.ply` files into the viewer, or use the Import button.
4.  **Editing**:
    *   Use the **Selection Tool** (Rectangle icon) to select points.
    *   Click **Delete** to remove selected points (syncs with server if streamed).
5.  **Visibility**: Toggle the "Eye" icon in the Layer list to show/hide Point Clouds or GNG topology.

### Topological Map Notes

- The viewer consumes `/topological_map` directly through the backend bridge node.
- The viewer consumes robot pose updates from `/viewer/internal/stream/robot` through the backend bridge node.
- When using `gng_safety`, `topoarm_viewer_bridge.launch.py` now includes the joint-state runtime, so one launch can prepare `/joint_states` and stream the arm pose.
- The current color convention for topology labels is:
  - `safe` = green
  - `danger` = yellow
  - `collision` = red
- The arm renderer follows `/joint_states` via the gng_safety bridge and updates the pose in real time.
- In the GNG Topology panel, `Nodes` and `Edges` can be toggled independently, and `Edge Width` adjusts the whole graph thickness.
