# ToPoFuzzy WebViewer システム要件定義書

## 1. システム概要

LiDARから得られる点群データおよびGNG（Growing Neural Gas）のノードエッジグラフをWebブラウザ上で可視化・解析するシステム。
静的ファイルの読み込みに加え、ROSドライバ経由でのリアルタイム表示に対応し、デジタルツイン、施工管理、セキュリティ監視等の用途を想定する。

## 2. 技術スタック・開発環境

* **Frontend Framework:** React + TypeScript + Vite
* **3D Engine:** Three.js (React Three Fiber)
* **Styling:** Tailwind CSS (推奨)
* **Communication:** WebSocket (ROS2 Bridge) / HTTP (File Upload)
* **Constraint:** 商用利用可能(MIT/Apache等)、軽量動作

## 3. 機能要件 (Functional Requirements)

機能を「データの入出力」「基本表示」「加工・編集」「解析・応用」の4つに分類します。

### 3.1 データの入出力 (Data I/O)

| 機能ID | 機能名 | 詳細要件 | UI担当 | ロジック担当 |
| --- | --- | --- | --- | --- |
| **IO-01** | **ファイル読込** | ドラッグ&ドロップまたはダイアログ選択による読込 。<br>対応形式: PC2, PCD, LAS, PLY, XML 。 | 読込エリア/ボタン作成 | パーサー実装 (Loader) |
| **IO-02** | **リアルタイム受信** | ROS LiDARドライバからのストリームデータをWebSocket経由で受信し描画 。 | 接続ステータス表示 | WebSocket接続/バッファ更新 |
| **IO-03** | **保存・書出** | 表示中の点群や解析結果をローカルファイルとして保存。座標変換後の保存も含む 。

 | 保存ボタン/形式選択 | シリアライズ/ダウンロード処理 |

### 3.2 基本表示・操作 (Basic Visualization)

| 機能ID | 機能名 | 詳細要件 | UI担当 | ロジック担当 |
| --- | --- | --- | --- | --- |
| **VZ-01** | **視点操作** | オービット（回転）、パン（平行移動）、ズーム 。プリセット視点（正面/上面/側面）へのワンタッチ切り替え 。| 操作ガイド/視点ボタン | Camera Controls設定 |
| **VZ-02** | **レイヤー管理** | 読み込んだファイルや抽出結果ごとの表示ON/OFF、透明度調整、表示順序変更 。 | レイヤーパネル実装 | レイヤー状態管理と描画反映 |

| **VZ-04** | **描画・注釈** | 画面上に円、四角形などの図形を描画し、注釈や範囲指定として利用 。 | ツールパレット | 2D/3D描画オブジェクト管理 |

### 3.3 加工・編集 (Manipulation)

| 機能ID | 機能名 | 詳細要件 | UI担当 | ロジック担当 |
| --- | --- | --- | --- | --- |
| **MN-01** | **平行移動・回転** | レイヤーごとの座標変換。数値入力および3Dギズモ（インタラクティブマーカー）による操作 。| 数値入力フォーム | Matrix変換処理/Gizmo実装 |

| **MN-05** | **差分表示** | 2つの点群（Before/After）の差分を計算し色分け表示。差分の定量データ（体積等）表示 。| 差分結果パネル | KD-Tree等による距離計算 |

### 3.4 解析・応用 (Advanced Analysis)

※これらは計算負荷が高いため、WebWorkerまたはバックエンド処理を想定します。

| 機能ID | 機能名 | 詳細要件 | UI担当 | ロジック担当 |
| --- | --- | --- | --- | --- |
| **AN-01** | **物体抽出** | 登録したテンプレート形状と一致する物体を抽出・別レイヤー化 。| テンプレート管理画面 | テンプレートマッチング処理 |

| **AN-05** | **行動予測・識別** | 移動体の未来位置予測（矢印表示）および種別（人/車）の識別 。| 予測・識別ラベル表示 | 予測モデル推論/表示連携 |

## 4. UI/UX 要件 (Layout & Interaction)

画面は大きく3つのセクションに分割して開発を進めます。

1. **メインビュー (Canvas Area)**
* 画面中央の大部分を占める。
* ここにThree.jsのCanvas (`<Canvas>`) を配置。
* 右上にFPSや点群数のオーバーレイ表示。


2. **サイドバー (Control Panel)**
* 画面左側（または右側）に配置。折りたたみ可能。
* **タブ切り替え:** 「レイヤー」「表示設定(ヒートマップ等)」「編集(移動・削除)」「解析」でパネルを切り替える。
* ファイルリスト（レイヤー一覧）はPhotoshopのようなリスト形式UIとする。


3. **ツールバー (Floating/Fixed Tools)**
* 画面下部または上部にアイコンで配置。
* 視点リセット、矩形選択モード、計測モードなどの頻繁に使うツールを配置。

## 5. データ構造設計案 (Shared Types)

分業の要となる型定義です。`src/types/` に配置します。

```typescript
// 基本的な点群データ構造
export type PointCloudData = {
  id: string;
  name: string;
  visible: boolean;
  opacity: number;
  matrix: THREE.Matrix4; // 位置・回転
  points: Float32Array;  // [x,y,z, x,y,z...]
  colors?: Float32Array; // [r,g,b, r,g,b...] (ヒートマップ用)
  intensities?: Float32Array; // 反射強度
};

// GNG/グラフデータ構造
export type GraphData = {
  id: string;
  nodes: Float32Array; // ノード座標
  edges: Uint32Array;  // エッジインデックス [src, tgt, src, tgt...]
  clusterLabels?: number[];
};

// 解析結果（バウンディングボックス等）
export type BoundingBox = {
  id: string;
  position: [number, number, number];
  dimensions: [number, number, number]; // width, height, depth
  label: string; // "Person", "Car" etc.
  confidence: number;
};

```

## 6. 開発フェーズ案

機能が多岐にわたるため、段階的な開発を推奨します。

* **Phase 1 (MVP: Minimum Viable Product):**
* ファイル読込 (IO-01)
* 点群・グラフの単純表示 (VZ-01, VZ-05)
* レイヤーON/OFF (VZ-02)


* **Phase 2 (Editor):**
* 平行移動・回転 (MN-01)
* ヒートマップ表示 (VZ-03)
* 不要部分除去 (MN-03)


* **Phase 3 (Real-time & Analysis):**
* WebSocket連携 (IO-02)
* 移動体検出・ゾーン監視等の解析機能 (AN-03, AN-04)
